## 2. 核心观察：三层结构（最终确认版）

从手机视频和图片中，我识别出 **三个视觉层**，现在结合图片进行最终确认：

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   ┌──────────────────────────────┐  ┌───┐                │
│   │                              │  │相机│ ← 第1层：原生刘海│
│   │        第2层：次级框            │  └───┘  （纯黑/最黑）  │
│   │     （淡黑色，比刘海更宽）        │                      │
│   │  ┌────────────────────────┐  │                      │
│   │  │       云朵图标区域       │  │                      │
│   │  └────────────────────────┘  │                      │
│   └──────────────────────────────┘                      │
│        ↑↑↑↑↑↑  第3层：蓝光（边缘发光）  ↑↑↑↑↑↑             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 各层特征（结合图片确认）

| 层级 | 颜色 | 行为 | 与其他层关系 |
|------|------|------|--------------|
| **1. 原生刘海** | 最黑（纯黑 `#000000`） | 一直存在，不变化 | 物理实体，苹果电脑的相机挖孔区域 |
| **2. 次级框** | 淡黑色（`opacity(0.75)` 左右） | 云朵靠近/拖拽时动态弹出，比原生刘海更宽 | 包围着原生刘海，形成扩展区域 |
| **3. 蓝光** | 淡蓝色（`#8CC5FF` 左右，透明度约 30%） | 沿次级框边缘发光 | 次级框的边缘光晕效果，不是独立层 |

### 关键理解（用户原话对应）

> "弹出来的框本质上就是一个黑色的框，跟原生的弹出框是一样的"

✅ **次级框**就是那个"弹出来的框"，比原生刘海更宽，淡黑色。

> "因为它会发光，我估计应该是刘海顶部的发卡，或者是靠近的时候，云发出了一个很亮的蓝光"

✅ **蓝光**是次级框边缘的发光效果。图片显示蓝光沿整个次级框边缘，不只是顶部——但"刘海顶部发卡"的猜测是合理的方向。

- 颜色是淡蓝色，透明度约 30%
- 触发时机：次级框弹出时同步出现

### 3.3 实现方案建议

基于图片观察，Invoko 的蓝光效果可以用以下 SwiftUI 代码实现：

```swift
// 次级框 + 蓝色边缘光晕
ZStack {
    // 次级框主体：淡黑色
    RoundedRectangle(cornerRadius: 14)
        .fill(Color.black.opacity(0.75))
    
    // 蓝色内边缘描边（细线）
    RoundedRectangle(cornerRadius: 14)
        .stroke(Color(hex: "#8CC5FF").opacity(0.3), lineWidth: 1)
}
.overlay(
    // 外层蓝色光晕（模糊边缘）
    RoundedRectangle(cornerRadius: 16)
        .stroke(Color(hex: "#8CC5FF").opacity(0.15), lineWidth: 3)
        .blur(radius: 4)
)
.shadow(color: Color(hex: "#8CC5FF").opacity(0.2), radius: 6)
```

## 4. 交互流程推断

根据视频分析，我推断 Invoko 的刘海交互包含以下状态：

```
浮动态 (floating)
    │
    │ 云朵拖向刘海
    ▼
靠近预览 (dockPreview)  ← 次级框弹出 + 蓝光出现
    │
    │ 释放云朵
    ▼
停靠态 (docked)        ← 云朵被吸入，变成小图标
    │
    │ 双击/悬停
    ▼
展开态 (expanded)      ← 卡片从刘海下方长出
```

### 各状态视觉特征

| 状态 | 云朵 | 刘海 | 蓝光 | 次级框 |
|------|------|------|------|--------|
| 浮动态 | 正常大小，可自由移动 | 原生刘海 | 无 | 无 |
| 靠近预览 | 吸附到刘海位置 | 原生刘海 | 出现 | 弹出（淡黑） |
| 停靠态 | 缩小为小图标 | 扩展 + 云朵图标 | 持续显示 | 保持 |
| 展开态 | 隐藏/变为头部图标 | 扩展 + 卡片顶部融合 | 可能变化 | 扩展为卡片顶部 |

## 6. 假设总结（修正版）

基于图片复核，我对我的假设进行以下修正：

### ❌ 假设 A 刘海顶部发卡光效（修正前）

- 蓝光是一条**紧贴刘海顶部边缘的细线**
- 使用 `LinearGradient` + `blur` 实现

### ✅ 假设 A 修正：边缘光晕（edge glow）

- 蓝光不是单一方向的"发卡光"
- 而是**整个扩展刘海边缘的光晕效果**
- 包括：顶部、左侧、与相机交接处
- 更像是次级框边框的半透明描边 + 外层阴影

### ✅ 假设 B：次级框是刘海横向扩展

- 次级框是原生刘海的**横向扩展版本**
- 包含相机区域 + 云朵图标区域
- 颜色比原生刘海略淡（`opacity(0.75)` 左右）
- 圆角可能比原生刘海更大

### ✅ 假设 C：光效是次级框的边缘描边

- 蓝光不是独立的发光层
- 而是次级框的**边框渐变**效果
- 可用 `stroke` + 半透明蓝色 + 外层 `shadow` 实现

## 9. 结论（Gemini 3.1 Pro 最终版）

（本结论由 Gemini 3.1 Pro 基于图片复核后确认）

基于图片观察，Invoko 的刘海光效包含以下三个清晰元素：

1. **原生刘海**：纯黑（`#000000`），苹果 Mac 物理刘海/相机挖孔，一直存在
2. **次级框**：淡黑色（`opacity(0.75)`），比原生刘海更宽，包围相机区域，云靠近时动态弹出
3. **蓝光**：淡蓝色（`#8CC5FF`），沿整个次级框边缘发光，是边缘描边 + 外层光晕的组合效果

当前 PromptCocoPilot 使用的是大范围 RadialGradient 光晕，与 Invoko 的边缘描边效果形态不同。

### 复刻建议

如果要复刻 Invoko 的视觉效果，建议：

1. **用淡黑色 RoundedRectangle 替代 RadialGradient** 作为次级框主体
2. **用 `stroke` + `shadow` 组合** 实现蓝色边缘光晕
3. **动画参数参考**：`spring(response: 0.34, dampingFraction: 0.74)`

### 动态交互机制（四帧分析）

通过四帧关键帧分析，蓝光触发机制如下：

| 帧 | 云朵位置 | 蓝光效果 |
|----|----------|----------|
| 帧1 | 云朵在下方，未靠近 | ❌ 刘海区域没有蓝光（纯黑） |
| 帧2 | 云朵开始靠近 | ⚡ 云朵边缘开始发光，刘海有淡淡蓝光 |
| 帧3 | 云朵更接近 | ✨ 蓝光更亮 |
| 帧4 | 云朵完全在刘海下方 | 💫 蓝光最强 |

**核心机制**：蓝光不是刘海自己发的，而是云朵靠近时**触发激活**的效果。距离越近，蓝光越强。

```
云朵靠近 → 触发蓝光 → 越近越亮 → 完全重叠时最强
```

### 修正后的理解

| 元素 | 之前理解 | 修正后理解 |
|------|----------|------------|
| **蓝光形态** | 边缘描边（只有边缘发光） | **整体光晕**（整个区域都在发光） |
| **蓝光分布** | 边缘最亮，中心无光 | 边缘稍亮，**中心也明显发光** |
| **蓝光触发** | 刘海自己发光 | **云朵靠近触发**，刘海被"照亮" |
| **次级框填充色** | `opacity(0.75)` 淡黑色 | `opacity(0.9)` 深黑色 + 蓝光叠加 |
| **"淡化"的原因** | 填充色本身是淡黑色 | 蓝光渗透 + 向内扩散 |
| **蓝光角色** | 边缘装饰效果 | **交互反馈元素**（响应云朵距离） |

### 实现方案（动态交互版）

```swift
// 动态蓝光效果：响应云朵距离
struct NotchGlowView: View {
    let cloudProximity: CGFloat  // 0.0 = 远离, 1.0 = 完全重叠
    
    var body: some View {
        ZStack {
            // 1. 深色填充基础
            RoundedRectangle(cornerRadius: 14)
                .fill(Color.black.opacity(0.9))
            
            // 2. 整体蓝光叠加（强度随距离变化）
            Color(hex: "#8CC5FF")
                .opacity(cloudProximity * 0.25)  // 距离越近，蓝色越亮
                .blur(radius: 20)
            
            // 3. 更亮的中心光斑
            RadialGradient(
                colors: [
                    Color(hex: "#8CC5FF").opacity(cloudProximity * 0.4),
                    .clear
                ],
                center: .center,
                startRadius: 0,
                endRadius: 100
            )
            
            // 4. 边缘加强光
            RoundedRectangle(cornerRadius: 14)
                .stroke(
                    Color(hex: "#8CC5FF").opacity(cloudProximity * 0.6),
                    lineWidth: 2
                )
        }
        .shadow(
            color: Color(hex: "#8CC5FF").opacity(cloudProximity * 0.5),
            radius: 15
        )
    }
}
```

**关键逻辑**：蓝光强度由 `cloudProximity` 控制——云朵越近，数值越大，蓝光越亮。
