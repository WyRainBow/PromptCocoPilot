- **双轨运行时**：模块包含 Python (pywebview) 和 Swift (Native AppKit) 两套并行的 UI 实现，均作为独立进程运行。
- **共享数据契约**：所有 UI 变体均直接读取 ~/.claude/sessions 和 projects 下的 JSON/JSONL 文件以获取会话上下文，实现了 UI 层与 Claude Code 核心数据的解耦。
- **外部服务依赖**：UI 仅负责交互与上下文提取，实际的 LLM 增强逻辑委托给外部 HTTP 服务 (mcp-server/http_server.py)，启动时自动探测并拉起该服务。
- **统一入口模式**：Python 端通过 bin/ 脚本动态注入 src/ 路径并启动；Swift 端通过 @main 入口初始化全局热键与无障碍窗口控制器。
