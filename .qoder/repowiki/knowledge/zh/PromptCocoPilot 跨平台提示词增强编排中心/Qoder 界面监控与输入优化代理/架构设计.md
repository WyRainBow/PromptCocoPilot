- **入口与生命周期**：`bin/qoder-optimize-input.js` 作为 CLI 入口，支持 `install-agent` / `uninstall-agent` 管理 macOS LaunchAgent，或直接运行 `runDaemon` 启动守护进程。
- **核心守护逻辑**：`src/daemon.js` 负责连接 Qoder 的 DevTools 端口（从 `DevToolsActivePort` 读取），定位 Agents Window 目标，并通过 WebSocket 维持长连接。
- **通信桥接**：利用 `Runtime.addBinding` 建立 JS 到 Node.js 的双向通信通道，监听 `optimize-request` 事件并转发至本地增强服务（默认 `localhost:8765`）。
- **页面注入**：`src/observer_script.js` 生成并在目标页面执行注入脚本，通过 DOM 观察器在输入框旁动态挂载“优化输入”按钮，拦截用户输入并触发后端处理。
- **基础设施**：`src/devtools_client.js` 封装了简化的 CDP (Chrome DevTools Protocol) 客户端；`src/launch_agent.js` 处理 macOS 特有的 `launchctl` 集成与 plist 配置生成。
