- **双轨并行架构**：模块维护 Python (pywebview) 和 Swift (AppKit/SwiftUI) 两套独立的 UI 进程，均作为系统级悬浮组件运行，互不干扰但功能对等。
- **文件系统共享契约**：Python (`session_reader.py`) 与 Swift (`SessionReader.swift`) 实现了完全对称的会话解析逻辑，直接读取 `~/.claude/sessions` 和 `projects` 下的 JSON/JSONL 文件，确立了以文件系统为中介的跨语言状态同步机制。
- **外部服务解耦**：UI 层仅负责上下文提取与交互展示，核心的 LLM 增强逻辑委托给外部 HTTP 服务；所有 UI 变体在启动时均执行 `_ensure_server_ready` 或等效逻辑，自动探测并拉起后端服务。
- **统一的路径注入与入口**：Python 端通过 `bin/` 脚本动态注入 `src/` 路径解决模块导入问题，Swift 端则通过 `@main` 初始化全局热键与无障碍控制器，形成了标准化的独立进程启动模式。
