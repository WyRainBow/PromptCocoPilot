- **入口层**：`bin/codex-optimize-input.js` 提供 CLI 接口，支持启动调试模式 (`launch`)、DOM 探测 (`probe`) 以及 macOS LaunchAgent 的注册与卸载。
- **核心逻辑层**：`src/daemon.js` 负责维护 WebSocket 长连接，通过 `discoverDevToolsPort` 动态发现调试端口，并利用 `Runtime.addBinding` 和脚本注入实现页面交互。
- **协议适配层**：`src/devtools_client.js` 封装了 CDP (Chrome DevTools Protocol) 的底层通信，处理消息 ID 映射与事件分发。
- **注入脚本层**：`src/observer_script.js` 生成在渲染进程中执行的 JavaScript，负责定位输入框（支持 ProseMirror/Textarea）、注入“优化输入”按钮并处理剪贴板事件。
- **基础设施层**：`src/codex_launcher.js` 和 `src/codex_paths.js` 处理 macOS 下的应用进程管理、端口文件读取及路径配置；`src/launch_agent.js` 负责生成并管理 `launchctl` 配置文件以实现开机自启。
