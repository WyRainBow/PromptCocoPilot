- **双轨并行与对等契约**：模块维护两套独立的 UI 进程（Python `bin/` 脚本与 Swift `@main` 入口），均作为系统级非激活（non-activating）悬浮组件运行。两者通过对称的 `SessionReader` 逻辑直接读取 `~/.claude/sessions` 及 `projects` 下的 JSONL 文件，确立了以文件系统为中介的跨语言状态同步机制。
- **外部服务解耦与自动拉起**：UI 层仅负责上下文提取与交互展示，核心的 LLM 增强逻辑委托给外部 HTTP 服务；所有 UI 变体在启动时均执行 `_ensure_server_ready` 或等效探测逻辑，自动在后台拉起后端服务。
- **统一的路径注入与入口模式**：Python 端通过 `sys.path.insert` 动态注入 `src/` 路径解决模块导入问题，Swift 端则通过 `AppDelegate` 初始化全局热键与无障碍控制器，形成了标准化的独立进程启动模式。
